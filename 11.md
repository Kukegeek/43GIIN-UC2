Actúa como un desarrollador Full-Stack Arquitecto experto en Next.js (App Router), Supabase, Vercel, GitHub Actions y la API de AliExpress (AliExpress Open Platform / Affiliate API).

Quiero construir una plataforma de afiliados de AliExpress desde cero. La web funcionará como un ecommerce avanzado alojado en Vercel, pero toda la automatización pesada (actualización de precios e importación automática) se ejecutará fuera de Vercel, utilizando GitHub Actions para evitar límites de tiempo (timeouts).

Stack Tecnológico:

Frontend y Servidor Web: Next.js (App Router), React, Tailwind CSS, Recharts (gráficos). Alojado en Vercel.

Base de datos y Autenticación: Supabase (PostgreSQL y Supabase Auth).

Automatización (Cron Jobs): GitHub Actions (ejecutando scripts de Node.js independientes dentro del repositorio).

Requisitos Core y Lógica de Negocio:

1. Panel de Administración Privado (/admin):

Gestor de Categorías (CRUD) para crear mis propias secciones simplificadas.

Buscador manual de la API de AliExpress para importar productos puntuales a mi base de datos.

Configuración del Auto-Importer: Una vista donde yo pueda guardar "Palabras Clave" o "IDs de Categoría de AliExpress" asociadas a mis categorías propias, para que el bot sepa qué buscar.

2. Automatización con GitHub Actions (El Motor Oculto):
Necesito que crees la carpeta .github/workflows y los scripts necesarios en Node.js para dos procesos:

Worker 1 (Auto-Importer): Se ejecuta 1 vez al día. Lee las palabras clave configuradas, busca los productos más vendidos en la API de AliExpress y guarda automáticamente los nuevos en Supabase.

Worker 2 (Price Updater & History): Se ejecuta cada 72 horas. Extrae todos los productos de Supabase y consulta sus precios actuales en la API. Crítico: El script debe tener un delay de 500ms entre cada consulta a la API para no exceder el límite de concurrencia de AliExpress (5 peticiones/segundo).

Historial: Si el Price Updater detecta un cambio de precio, inserta un registro en la tabla price_history (fecha y precio).

3. Interfaz de Usuario (Storefront en Vercel):

Autenticación: Login/Registro con Supabase Auth.

Lista de Seguimiento (Watchlist): Botón de "Añadir a seguimiento" y vista /mi-cuenta para usuarios registrados.

Home Dinámica: Novedades, Mayores Descuentos (calculado con el historial), y Más Vendidos (registrar cada clic saliente en la BD).

Página de Producto Individual:

Título, imagen, precio actual, % descuento, etiqueta "Actualizado hace XX horas".

Gráfico de historial de precios (Recharts).

Botón "Comprar en AliExpress" con mi Tracking ID.

4. Geolocalización (Vercel Edge):

Usar proxy.ts (o el equivalente en Next.js 16) y la cabecera x-vercel-ip-country para detectar el país. Pasar este dato a la API (ship_to_country) para filtrar envíos.

Plan de Ejecución (Paso a paso, espera mi confirmación tras cada paso):

Arquitectura DB: Código SQL para Supabase (Tablas: categories, products, price_history, users, watchlist, clicks_tracking, auto_importer_config) y políticas RLS.

Setup Base: Comandos de terminal para Next.js 16, dependencias y variables de entorno.

Automatización: Los archivos YAML de GitHub Actions y los scripts de Node.js (con el delay de 500ms y la firma de la API).

Backend/Admin: Rutas privadas, CRUD de categorías y buscador.

Frontend: UI de la tienda, gráficos Recharts, Watchlist y Geo-IP.
