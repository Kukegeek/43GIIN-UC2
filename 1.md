Actúa como un desarrollador Full-Stack Arquitecto experto en Next.js (App Router), Supabase, Vercel y la API de AliExpress (AliExpress Open Platform / Affiliate API).

Quiero construir una plataforma de afiliados de AliExpress desde cero, diseñada como un ecommerce avanzado con herramientas de seguimiento de precios para retener a los usuarios.

Stack Tecnológico:

Frontend: Next.js (App Router), React, Tailwind CSS. Gráficos con Recharts o similar.

Backend/Hosting: Vercel (Cron Jobs y Vercel Edge/Geo-IP headers).

Base de datos y Autenticación: Supabase (PostgreSQL y Supabase Auth).

Requisitos Core y Lógica de Negocio:

1. Panel de Administración Privado (/admin):

Buscador API: Vista para buscar productos en AliExpress, ordenables por precio y ventas. Checkbox para seleccionar e "Importar a la tienda".

Gestor de Categorías Personalizadas: No quiero replicar el caos de categorías de AliExpress. Necesito un CRUD (Crear, Leer, Actualizar, Eliminar) para crear mis propias categorías simplificadas. Al importar un producto, lo asignaré a una de mis categorías personalizadas.

2. Sistema de Actualización e Historial de Precios (Cron Job):

Un Vercel Cron Job que actualice los productos de mi Supabase en lotes cada 72 horas para no exceder los límites de la API.

Historial: Cada vez que el Cron detecte un cambio de precio, no solo debe actualizar el precio actual en la tabla products, sino insertar un nuevo registro en una tabla price_history (con fecha y precio) para trazar la evolución.

3. Interfaz de Usuario (Storefront):

Autenticación: Login/Registro básico con Supabase Auth.

Lista de Seguimiento (Watchlist): Los usuarios logueados deben tener un botón de "Añadir a seguimiento" en cada producto. Una vista privada (/mi-cuenta) donde vean los productos que están siguiendo.

Home Dinámica: Tres secciones: 1) Novedades (últimos añadidos), 2) Mayores Descuentos (calculado sobre su historial), 3) Más Vendidos (productos con más clics en el botón de compra; registrar clics en BD).

Página de Producto Individual:

Título, imagen, precio actual, % de descuento, "Actualizado hace XX horas" y "Envío Gratis".

Gráfico de Historial de Precios: Una representación visual (línea temporal) sacada de la tabla price_history para que el usuario vea si es un buen momento para comprar.

Botón de compra: "Comprar en AliExpress" (con mi Tracking ID de afiliado).

4. Geolocalización de Usuarios:

Utilizar cabeceras de Vercel (x-vercel-ip-country) para detectar el país del visitante. Usar este dato en la API de AliExpress (ship_to_country) para mostrar productos con envío a su país.

Plan de Ejecución (Paso a paso, espera mi confirmación tras cada paso):

Arquitectura DB: Dame el código SQL para Supabase (Tablas: categories, products, price_history, users, watchlist, clicks_tracking) y políticas RLS de seguridad.

Setup: Comandos de terminal, dependencias y .env.local.

Backend/Admin: Integración con la API de AliExpress, CRUD de categorías y buscador para importar.

Cron Job: Script de actualización e inserción en el historial de precios.

Frontend/Storefront: UI, Auth, Home, Producto (con gráfico de precios) y Lista de Seguimiento.
